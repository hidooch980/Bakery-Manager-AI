- این سرویس قبلاً **stub** بود و همیشه صفر برمی‌گرداند (`sales = 0, cost = 0`)!
- حالا مقادیر واقعی از دیتابیس محاسبه می‌شود:
  - **فروش کل** = مجموع فروش‌ها (`Sale.total`) + فروش ثبت‌شده در تولید (`ProductionBatch.totalSale`)
  - **هزینه کل** = مجموع هزینه‌ها (`Expense.amount`)
  - **سود** = فروش کل − هزینه کل
  - به‌علاوه تفکیک (breakdown): فروش نقدی/کارتی، تعداد نان، کیسه آرد مصرفی

## ✅ فرانت‌اند — صفحات جدید

| صفحه | مسیر | امکانات | Endpointها |
|---|---|---|---|
| محصولات | `/products` | افزودن محصول (نام + قیمت، دسته خودکار «عمومی») + لیست با موجودی و دسته | `GET/POST /products` |
| کارکنان | `/employees` | افزودن کارمند (نام، نقش فارسی، موبایل، حقوق پایه) + لیست کارکنان | `GET/POST /employees` |
| تحلیل و سود | `/analytics` | کارت‌های فروش/هزینه/سود + تفکیک جزئیات + توصیه‌های مشاور هوشمند | `GET /profit-dashboard`، `GET /ai-advisor` |

- نقش‌های کارکنان با برچسب فارسی نمایش داده می‌شوند (فروشنده، خمیرگیر، چانه‌گیر، نانوا، حسابدار، مدیر).
- اعداد صفحه تحلیل با فرمت فارسی (`toLocaleString("fa-IR")`) نمایش داده می‌شوند.
- صفحه کارکنان طبق RolesGuard فقط برای MANAGER (و ADMIN) قابل دسترسی است.

## ▶️ تست سریع

```bash
docker compose up --build
# http://localhost:5173 — ورود: 09000000000 / Admin@123
```

سناریوی پیشنهادی تست:
1. در صفحه محصولات یک محصول بسازید (مثلاً نان تافتون، قیمت ۵۰۰۰)
2. در صفحه تولید یک تولید ثبت کنید
3. در صفحه هزینه‌ها یک هزینه ثبت کنید
4. صفحه «تحلیل و سود» را باز کنید — اعداد واقعی و توصیه مشاهده می‌شود
