## ✅ بک‌اند — تکمیل چرخه احراز هویت

تا قبل از این فاز، refresh token فقط در دیتابیس ذخیره می‌شد ولی هیچ endpointی برای استفاده از آن وجود نداشت!

### Endpointهای جدید (`backend/src/auth/`)
| Endpoint | وظیفه |
|---|---|
| `POST /auth/refresh` | تمدید توکن دسترسی با refresh token (با rotation: توکن قبلی باطل و توکن جدید صادر می‌شود) |
| `POST /auth/logout` | باطل کردن refresh token در سمت سرور |
| `GET /auth/me` | دریافت اطلاعات کاربر جاری از روی توکن |

- توکن‌های منقضی هنگام استفاده حذف می‌شوند.
- منطق صدور توکن در متد مشترک `issueTokens` یکپارچه شد.

## ✅ فرانت‌اند — صفحات عملیاتی جدید

### کامپوننت جدید: `src/components/Layout.tsx`
- منوی کناری مشترک برای همه صفحات + نمایش نام/نقش کاربر + دکمه خروج

### صفحات جدید (همه متصل به API واقعی)
| صفحه | مسیر | امکانات | Endpointها |
|---|---|---|---|
| فروش | `/sales` | ثبت فروش چندقلمی با انتخاب محصول + لیست فروش‌ها | `GET/POST /sales` ، `GET /products` |
| تولید | `/production` | ثبت تولید (شیفت، کیسه آرد، وزن، چانه، نان) + لیست تولیدات | `GET/POST /production` |
| هزینه‌ها | `/expenses` | ثبت هزینه (عنوان، مبلغ، دسته، یادداشت) + لیست هزینه‌ها | `GET/POST /expenses` |
| گزارش روزانه | `/report` | صفحه Dashboard قبلاً بلااستفاده بود، حالا متصل شد | `GET /daily-report` |

- فرم‌ها دقیقاً مطابق DTOها و مدل‌های Prisma بک‌اند طراحی شده‌اند (مثلاً `items[{productId, quantity}]` برای فروش).
- نقش ADMIN طبق RolesGuard به همه مسیرها دسترسی دارد؛ سایر نقش‌ها محدودیت خود را دارند.
